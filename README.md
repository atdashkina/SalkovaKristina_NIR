# Проект: Персонализированная система развивающих заданий для дошкольников

## Описание проекта

Данный проект направлен на создание системы персонализации образовательного процесса для детей дошкольного возраста (3–7 лет) с использованием методов машинного обучения. На основе результатов тестирования по 50 диагностическим заданиям (категории: «Формы», «Цвета», «Счёт», «Память», «Логика») система автоматически определяет уровень развития ребёнка и формирует индивидуальный план развивающих заданий.

### Основные этапы

1. **Генерация данных** – создание синтетического датасета с результатами тестирования 300 детей, метаданных заданий и банка развивающих заданий (4 уровня × 5 категорий).
2. **Предобработка и инженерия признаков** – агрегирование баллов по категориям, нормализация.
3. **Кластеризация (K‑Means)** – выделение 4 групп детей по уровню развития (от «требует поддержки» до «продвинутый»).
4. **Интерпретация кластеров** – построение когнитивных профилей и анализ сильных/слабых категорий.
5. **Генератор персонализированных заданий** – выдача 20 заданий (с упором на слабые категории) на основе обученной модели.

### Результаты

- **4 кластера** с удовлетворительным качеством (Silhouette ≈ 0.26).
- Банк из **160 заданий** (по 40 на каждый уровень).
- Работающий генератор, демонстрирующий персонализированные планы.

## Инструкция по запуску

### 1. Установка зависимостей

Рекомендуется использовать виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate         # Windows
```

Установите необходимые библиотеки:

```bash
pip install numpy pandas matplotlib seaborn scikit-learn openpyxl
```

### 2. Генерация данных

Запустите `generate_data.ipynb` (выполните все ячейки).  
Будут созданы три Excel-файла в той же директории.

### 3. Основной анализ и генерация заданий

Запустите ноутбук `preschool_ml_personalization.ipynb`:

- Он загрузит сгенерированные файлы.
- Выполнит EDA, кластеризацию, построит профили.
- Создаст объект `PersonalizedTaskGenerator`.
- Продемонстрирует генерацию планов для выбранных детей (в том числе для нового ребёнка).

### 4. Использование генератора отдельно

Вы можете импортировать класс `PersonalizedTaskGenerator`.  
Для применения в своём коде необходимо иметь обученную модель KMeans, scaler и банк заданий.

Пример вызова (после обучения):

```python
plan = generator.generate_plan(
    child_features=[5, 35.0, 45.0, 28.0, 30.0, 20.0],
    child_id="НОВЫЙ_РЕБЁНОК",
    n_tasks=20
)
generator.display_plan(plan)
```
