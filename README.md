# Personalized Preschool Task Recommendation System

Система персонализации развивающих заданий для детей дошкольного возраста на основе методов машинного обучения.

Проект моделирует полный цикл работы с образовательными данными:
- генерацию синтетического датасета результатов тестирования детей;
- формирование метаданных диагностических заданий;
- создание банка развивающих упражнений;
- кластеризацию детей по уровню когнитивного развития;
- генерацию персонализированного плана заданий для каждого ребёнка.

## О проекте

В проекте используется выборка из **300 детей** в возрасте **3–7 лет**, прошедших диагностический тест из **50 заданий** по пяти когнитивным категориям:
- формы;
- цвета;
- счёт;
- память;
- логика.

На основе результатов тестирования рассчитываются агрегированные признаки, после чего дети разделяются на **4 уровня развития** с помощью алгоритма **K-Means**. Для каждого уровня формируется набор развивающих заданий из заранее подготовленного банка, а итоговый генератор создаёт **индивидуальный план из 20 заданий** с акцентом на слабые стороны ребёнка.

## Основные возможности

- генерация синтетических данных тестирования;
- автоматическое сохранение промежуточных данных в Excel;
- разведочный анализ данных (EDA);
- нормализация признаков и подготовка данных к кластеризации;
- подбор количества кластеров (elbow method + silhouette analysis);
- кластеризация детей по уровню развития;
- визуализация кластеров с помощью **PCA** и **t-SNE**;
- интерпретация и профилирование кластеров;
- генерация персонализированных развивающих заданий;
- демонстрация работы модели как на существующих, так и на новых детях.

## Логика работы проекта

### 1. Генерация данных
Ноутбук `generate_data.ipynb` создаёт три файла:

- `children_data.xlsx` — результаты тестирования **300 детей**;
- `question_meta.xlsx` — описание **50 диагностических заданий**;
- `task_bank.xlsx` — банк из **160 развивающих заданий**.

### 2. Обработка данных
Основной ноутбук `preschool_ml_personalization.ipynb`:
- загружает Excel-файлы;
- рассчитывает групповые показатели по категориям;
- масштабирует признаки;
- подбирает число кластеров;
- обучает модель кластеризации;
- визуализирует результат;
- создаёт персональные планы заданий.

### 3. Персонализация
Для каждого ребёнка система:
1. принимает вектор признаков `[возраст, формы, цвета, счёт, память, логика]`;
2. определяет уровень развития через обученную модель;
3. выбирает задания из банка соответствующего уровня;
4. добавляет больший приоритет категориям с самыми низкими баллами;
5. возвращает персональный план из 20 заданий.

## Категории диагностики

Проект использует 5 когнитивных направлений:

| Категория | Количество заданий | Тип ответа |
|---|---:|---|
| Формы | 10 | бинарный (0/1) |
| Цвета | 10 | бинарный (0/1) |
| Счёт | 10 | балльный (0–3) |
| Память | 10 | бинарный (0/1) |
| Логика | 10 | бинарный (0/1) |

## Уровни развития

После кластеризации дети распределяются по 4 группам:

- **А — Требует поддержки**
- **Б — Средний**
- **В — Уверенный**
- **Г — Продвинутый**

Пример распределения в текущем эксперименте:
- А — 65 детей (**21.7%**)
- Б — 56 детей (**18.7%**)
- В — 99 детей (**33.0%**)
- Г — 80 детей (**26.7%**)

## Ключевые результаты

По итогам эксперимента в ноутбуке получены следующие метрики качества кластеризации:

- **Silhouette Score:** `0.2608`
- **Davies-Bouldin Index:** `1.3258`
- **Calinski-Harabasz Index:** `254.4`

Дополнительно:
- первые 2 компоненты PCA объясняют **83.7%** дисперсии;
- банк заданий содержит **160 упражнений**;
- на одного ребёнка формируется **20 персонализированных заданий**.

- данные в текущем проекте **синтетические**, а не собранные в реальной образовательной среде;
